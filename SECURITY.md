# Security Policy

## Supported versions

This repository contains operational runbooks for Velnor V1. Security-relevant
runbooks (incident response, access procedures) are reviewed quarterly or after
any SEV-1 incident.

## Reporting a vulnerability

Do **not** open a public GitHub issue for security vulnerabilities.

Contact the security team at: **security@velnor.com**

Include:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact assessment

We aim to acknowledge reports within **48 hours** and provide a resolution
timeline within **5 business days**.

## Runbook confidentiality

Some runbooks may contain references to internal endpoints, escalation contacts,
or access patterns. Do not share runbook URLs externally without confirming with
the Engineering lead that the content is safe for public viewing.

GitHub Pages publication is limited to non-sensitive operational procedures.
Incident response runbooks with sensitive details should be stored in the
internal wiki, not in this repository.
